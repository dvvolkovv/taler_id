# OAuth UI Kit — Phase 2: JavaScript SDK — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `@taler-id/oauth-client@0.1.0` to npm — a TypeScript SDK that lets browser SPAs add "Sign in with Taler ID" in ~5 lines of config plus an opt-in React adapter.

**Architecture:** New repo `taler-id-sdk-js` containing one npm package with two ESM/CJS entry points (`@taler-id/oauth-client` core + `@taler-id/oauth-client/react` adapter). Browser-only redirect-flow PKCE, sessionStorage default, on-demand token refresh, single `TalerIdAuthError` with `code` discriminator, Vitest+jsdom tests, GitHub Actions CI/release.

**Tech Stack:** TypeScript 5.x · `tsup` (bundler, ESM+CJS+types) · Vitest + jsdom + `@testing-library/react` · ESLint 9 flat config · GitHub Actions · `pnpm` for repo dependency management (npm consumers consume the published artifact).

**Spec:** `~/taler-id/docs/superpowers/specs/2026-04-29-oauth-ui-kit-phase-2-js-sdk.md`

**Working directory:** New repo at `~/taler-id-sdk-js/` (does not exist yet — created in Task 1). Do NOT do work inside `~/taler-id` — that's the backend, an entirely different repository.

**Backend integration target:** All OAuth endpoints used by tests and examples live at `https://id.taler.tirol/oauth/*`. Staging mirror at `https://staging.id.taler.tirol/oauth/*`. The pre-existing `taler-id-demo` client (system client, no userId, registered on both DEV+PROD) is what `examples/react-vite/` and the manual e2e test use.

// src/pkce.ts
function base64urlEncode(bytes: Uint8Array): string {
  let str = '';
  for (const b of bytes) str += String.fromCharCode(b);
  return btoa(str).replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');
}

export function generateCodeVerifier(): string {
  const bytes = new Uint8Array(32);
  crypto.getRandomValues(bytes);
  return base64urlEncode(bytes);
}

export async function generateCodeChallenge(verifier: string): Promise<string> {
  const data = new TextEncoder().encode(verifier);
  const digest = await crypto.subtle.digest('SHA-256', data);
  return base64urlEncode(new Uint8Array(digest));
}

export function generateState(): string {
  const bytes = new Uint8Array(16);
  crypto.getRandomValues(bytes);
  return base64urlEncode(bytes);
}

// src/storage.ts
export interface Storage {
  get(key: string): string | null;
  set(key: string, value: string): void;
  remove(key: string): void;
}

export type StorageKind = 'session' | 'local' | 'memory';

class WebStorageAdapter implements Storage {
  constructor(private readonly underlying: globalThis.Storage) {}
  get(key: string) { return this.underlying.getItem(key); }
  set(key: string, value: string) { this.underlying.setItem(key, value); }
  remove(key: string) { this.underlying.removeItem(key); }
}

class MemoryStorageAdapter implements Storage {
  private readonly map = new Map<string, string>();
  get(key: string) { return this.map.get(key) ?? null; }
  set(key: string, value: string) { this.map.set(key, value); }
  remove(key: string) { this.map.delete(key); }
}

export function createStorage(kind: StorageKind): Storage {
  switch (kind) {
    case 'session': return new WebStorageAdapter(sessionStorage);
    case 'local': return new WebStorageAdapter(localStorage);
    case 'memory': return new MemoryStorageAdapter();
  }
}

// src/state.ts
export interface UserInfo {
  readonly sub: string;
  readonly [claim: string]: unknown;
}

export interface AuthState {
  readonly user: UserInfo | null;
  readonly isAuthenticated: boolean;
  readonly isLoading: boolean;
}

export type AuthStateListener = (state: AuthState) => void;

export class AuthStateEmitter {
  private state: AuthState;
  private readonly listeners = new Set<AuthStateListener>();

  constructor(initial: AuthState) {
    this.state = initial;
  }

  get(): AuthState {
    return this.state;
  }

  set(next: AuthState): void {
    this.state = next;
    for (const listener of this.listeners) listener(next);
  }

  subscribe(listener: AuthStateListener): () => void {
    this.listeners.add(listener);
    return () => { this.listeners.delete(listener); };
  }
}
```

import { createStorage, type Storage, type StorageKind } from './storage';
import { AuthStateEmitter, type AuthState, type AuthStateListener, type UserInfo } from './state';

export interface TalerIdClientConfig {
  clientId: string;
  redirectUri: string;
  scope?: string;
  storage?: StorageKind;
  issuer?: string;
  onLog?: (level: 'debug' | 'info' | 'warn' | 'error', message: string, meta?: unknown) => void;
}

const DEFAULT_ISSUER = 'https://id.taler.tirol/oauth';
const DEFAULT_SCOPE = 'openid profile email';
const KEY_PREFIX = 'talerid:';
const K_ACCESS = `${KEY_PREFIX}access_token`;
const K_REFRESH = `${KEY_PREFIX}refresh_token`;
const K_ID = `${KEY_PREFIX}id_token`;
const K_EXPIRES = `${KEY_PREFIX}expires_at`;
const K_USER = `${KEY_PREFIX}user`;
const K_VERIFIER = `${KEY_PREFIX}pkce_verifier`;
const K_STATE = `${KEY_PREFIX}oauth_state`;

export class TalerIdClient {
  // Exposed for internal/testing inspection — not part of the documented API.
  readonly clientId: string;
  readonly redirectUri: string;
  readonly scope: string;
  readonly issuer: string;
  protected readonly storage: Storage;
  protected readonly emitter: AuthStateEmitter;
  protected readonly log: NonNullable<TalerIdClientConfig['onLog']>;

  constructor(config: TalerIdClientConfig) {
    if (!config.clientId) throw new TalerIdAuthError({ code: 'config', message: 'clientId is required' });
    if (!config.redirectUri) throw new TalerIdAuthError({ code: 'config', message: 'redirectUri is required' });
    this.clientId = config.clientId;
    this.redirectUri = config.redirectUri;
    this.scope = config.scope ?? DEFAULT_SCOPE;
    this.issuer = config.issuer ?? DEFAULT_ISSUER;
    this.storage = createStorage(config.storage ?? 'session');
    this.log = config.onLog ?? (() => {});
    const userJson = this.storage.get(K_USER);
    const initial: AuthState = {
      user: userJson ? (JSON.parse(userJson) as UserInfo) : null,
      isAuthenticated: this.isAuthenticated(),
      isLoading: false,
    };
    this.emitter = new AuthStateEmitter(initial);
  }

  isAuthenticated(): boolean {
    const token = this.storage.get(K_ACCESS);
    const expiresAtRaw = this.storage.get(K_EXPIRES);
    if (!token || !expiresAtRaw) return false;
    const expiresAt = Number(expiresAtRaw);
    return Number.isFinite(expiresAt) && expiresAt > Date.now();
  }

  onAuthStateChange(listener: AuthStateListener): () => void {
    return this.emitter.subscribe(listener);
  }

  async handleRedirectCallback(): Promise<boolean> {
    const params = new URLSearchParams(window.location.search);
    const code = params.get('code');
    if (!code) return false;

    const expectedState = this.storage.get(K_STATE);
    const actualState = params.get('state');
    if (!expectedState || !actualState || expectedState !== actualState) {
      throw new TalerIdAuthError({ code: 'state_mismatch', message: 'OAuth state parameter does not match' });
    }

    const verifier = this.storage.get(K_VERIFIER);
    if (!verifier) {
      throw new TalerIdAuthError({ code: 'state_mismatch', message: 'Missing PKCE verifier' });
    }

    const body = new URLSearchParams();
    body.set('grant_type', 'authorization_code');
    body.set('code', code);
    body.set('code_verifier', verifier);
    body.set('client_id', this.clientId);
    body.set('redirect_uri', this.redirectUri);

    let response: Response;
    try {
      response = await fetch(`${this.issuer}/token`, {
        method: 'POST',
        headers: { 'content-type': 'application/x-www-form-urlencoded' },
        body,
      });
    } catch (err) {
      throw new TalerIdAuthError({ code: 'network', message: 'Token request failed', cause: err });
    }
    if (!response.ok) {
      throw new TalerIdAuthError({ code: 'invalid_grant', message: `Token endpoint returned ${response.status}`, cause: await response.text() });
    }
    const json = await response.json() as { access_token: string; refresh_token?: string; id_token?: string; expires_in: number };
    this.storage.set(K_ACCESS, json.access_token);
    if (json.refresh_token) this.storage.set(K_REFRESH, json.refresh_token);
    if (json.id_token) this.storage.set(K_ID, json.id_token);
    this.storage.set(K_EXPIRES, String(Date.now() + json.expires_in * 1000));
    this.storage.remove(K_VERIFIER);
    this.storage.remove(K_STATE);

    // Clean ?code=&state= from the URL.
    const cleanUrl = window.location.origin + window.location.pathname;
    history.replaceState({}, document.title, cleanUrl);

    this.emitter.set({ user: this.emitter.get().user, isAuthenticated: true, isLoading: false });
    return true;
  }

  async getAccessToken(): Promise<string> {

    const expiresAtRaw = this.storage.get(K_EXPIRES);

  private refreshAccessToken(): Promise<string> {
    if (this.refreshInFlight) return this.refreshInFlight;
    const refreshToken = this.storage.get(K_REFRESH);
    if (!refreshToken) {
      this.clearSession();
      return Promise.reject(new TalerIdAuthError({ code: 'login_required', message: 'No refresh token' }));
    }
    const body = new URLSearchParams();
    body.set('grant_type', 'refresh_token');

        throw new TalerIdAuthError({ code: 'login_required', message: `Refresh returned ${response.status}` });
      }
      const json = await response.json() as { access_token: string; refresh_token?: string; id_token?: string; expires_in: number };
      this.storage.set(K_ACCESS, json.access_token);
      if (json.refresh_token) this.storage.set(K_REFRESH, json.refresh_token);
      if (json.id_token) this.storage.set(K_ID, json.id_token);
      this.storage.set(K_EXPIRES, String(Date.now() + json.expires_in * 1000));
      return json.access_token;
    })();

    this.refreshInFlight.finally(() => { this.refreshInFlight = null; });
    return this.refreshInFlight;
  }

  protected clearSession(): void {
    this.storage.remove(K_ACCESS);
    this.storage.remove(K_REFRESH);
    this.storage.remove(K_ID);
    this.storage.remove(K_EXPIRES);
    this.storage.remove(K_USER);
    this.emitter.set({ user: null, isAuthenticated: false, isLoading: false });
  }
```

  async logout(options: { returnTo?: string } = {}): Promise<void> {
    const access = this.storage.get(K_ACCESS);
    const refresh = this.storage.get(K_REFRESH);

    const revokeUrl = `${this.issuer}/token/revocation`;
    const revokes: Promise<unknown>[] = [];
    if (access) {
      const body = new URLSearchParams();
      body.set('token', access);
      body.set('token_type_hint', 'access_token');
      body.set('client_id', this.clientId);
      revokes.push(fetch(revokeUrl, { method: 'POST', headers: { 'content-type': 'application/x-www-form-urlencoded' }, body }).catch(() => {}));
    }
    if (refresh) {
      const body = new URLSearchParams();
      body.set('token', refresh);
      body.set('token_type_hint', 'refresh_token');
      body.set('client_id', this.clientId);
      revokes.push(fetch(revokeUrl, { method: 'POST', headers: { 'content-type': 'application/x-www-form-urlencoded' }, body }).catch(() => {}));
    }
    await Promise.all(revokes);

    this.clearSession();

    const endUrl = new URL(`${this.issuer}/session/end`);
    endUrl.searchParams.set('client_id', this.clientId);
    if (options.returnTo) endUrl.searchParams.set('post_logout_redirect_uri', options.returnTo);
    window.location.assign(endUrl.toString());
  }

import { type ReactNode, createContext, useEffect, useMemo, useState, useCallback } from 'react';
import { TalerIdClient, type TalerIdClientConfig } from '../client';
import { type AuthState } from '../state';
import { type TalerIdAuthValue } from './hook';

export const TalerIdContext = createContext<TalerIdAuthValue | null>(null);

export interface TalerIdProviderProps extends TalerIdClientConfig {
  children: ReactNode;
}

export function TalerIdProvider(props: TalerIdProviderProps) {
  const { children, ...config } = props;
  const client = useMemo(
    () => new TalerIdClient(config),
    // memoize on the four config keys that affect identity
    // eslint-disable-next-line react-hooks/exhaustive-deps
    [config.clientId, config.redirectUri, config.scope, config.storage, config.issuer],
  );

  const [state, setState] = useState<AuthState>(() => ({
    user: null,
    isAuthenticated: client.isAuthenticated(),
    isLoading: typeof window !== 'undefined' && new URLSearchParams(window.location.search).has('code'),
  }));

  useEffect(() => {
    const unsub = client.onAuthStateChange(setState);
    if (typeof window !== 'undefined' && new URLSearchParams(window.location.search).has('code')) {
      client.handleRedirectCallback()
        .then((handled) => {
          if (handled) return client.getUser();
          return null;
        })
        .catch(() => {
          setState((s) => ({ ...s, isLoading: false }));
        });
    }
    return unsub;
  }, [client]);

  const login = useCallback(() => client.loginWithRedirect(), [client]);
  const logout = useCallback((opts?: { returnTo?: string }) => client.logout(opts), [client]);
  const getAccessToken = useCallback(() => client.getAccessToken(), [client]);

  const value: TalerIdAuthValue = {
    user: state.user,
    isAuthenticated: state.isAuthenticated,
    isLoading: state.isLoading,
    login,
    logout,
    getAccessToken,
  };

  return <TalerIdContext.Provider value={value}>{children}</TalerIdContext.Provider>;
}

## Configuration

| Option        | Default                              | Description                                            |
| ------------- | ------------------------------------ | ------------------------------------------------------ |
| `clientId`    | (required)                           | OAuth client_id from your Taler ID registration        |
| `redirectUri` | (required)                           | Where Taler ID returns after login                     |
| `scope`       | `'openid profile email'`             | Space-separated scopes                                 |
| `storage`     | `'session'`                          | `'session'` \| `'local'` \| `'memory'`                 |
| `issuer`      | `'https://id.taler.tirol/oauth'`     | Override for staging                                   |
| `onLog`       | `() => {}`                           | `(level, message, meta) => void` for diagnostics       |

## API

| Method                                      | Description                                              |
| ------------------------------------------- | -------------------------------------------------------- |
| `await client.loginWithRedirect()`          | Start the OAuth flow (PKCE auto-generated)               |
| `await client.handleRedirectCallback()`     | On callback, exchange `?code=` for tokens. Returns `boolean` (false if no `?code=` in URL). |
| `await client.getUser()`                    | Fetch and cache `/oauth/me` userinfo                     |
| `await client.getAccessToken()`             | Returns access_token, auto-refreshing if near expiry     |
| `client.isAuthenticated()`                  | Sync, no network                                         |
| `client.onAuthStateChange(cb) → unsub`      | Subscribe to `{ user, isAuthenticated, isLoading }`      |
| `await client.logout({ returnTo? })`        | Revoke tokens, clear storage, redirect to `/oauth/session/end` |

## Errors

All errors are instances of `TalerIdAuthError` with a `code` field:

```ts
import { TalerIdAuthError } from '@taler-id/oauth-client';

try { await client.handleRedirectCallback(); }
catch (err) {
  if (err instanceof TalerIdAuthError && err.code === 'state_mismatch') {
    // CSRF / replayed link
  }
}
```

Codes: `login_required` · `consent_required` · `network` · `config` · `state_mismatch` · `invalid_grant`.

## Storage

By default, tokens live in `sessionStorage` (per-tab, cleared on tab close). Pass `storage: 'local'` for cross-tab persistence (longer-lived but bigger XSS blast radius), or `storage: 'memory'` for in-memory only (no persistence across reloads).

This task is the human-prerequisite gate. If `@taler-id` npm scope is not yet claimed and `NPM_TOKEN` is not yet in repo secrets, the workflow will fail at publish time. Both prerequisites are documented at the top of this plan.

- [ ] **Step 1: Verify prerequisites**

```bash
npm org ls @taler-id 2>&1 | head -5
gh secret list --repo dvvolkovv/taler-id-sdk-js
```

Expected:
- First command lists at least one user (Dmitry) — scope exists.
- Second command shows `NPM_TOKEN` in the list.

If either is missing, STOP and ask the human to complete the prerequisite before continuing.

- [ ] **Step 2: Confirm package name resolution one last time**

```bash
cd ~/taler-id-sdk-js
grep '"name":' package.json
npm view @taler-id/oauth-client 2>&1 | head -3
```

Expected:
- `package.json` says `"@taler-id/oauth-client"`.
- `npm view` returns `404 Not Found` (package does not yet exist; we're about to publish it).

If the package already exists with the same name and version, STOP — version `0.1.0` may have been published manually; coordinate with the human.

- [ ] **Step 3: Tag and push**

```bash
cd ~/taler-id-sdk-js
git tag -a v0.1.0 -m "Initial release of @taler-id/oauth-client"
git push origin v0.1.0
```

This triggers `.github/workflows/release.yml`.

- [ ] **Step 4: Watch the release run**

```bash
gh run watch
```

Expected: workflow proceeds through install → lint → typecheck → test → build → publish. Final step `npm publish --access public --provenance` succeeds with output containing `+ @taler-id/oauth-client@0.1.0`.

- [ ] **Step 5: Verify the package is live**

```bash
npm view @taler-id/oauth-client@0.1.0
```

Expected: returns the manifest with the correct `version`, `exports`, `repository`, and a `dist.signatures` block (provenance).

- [ ] **Step 6: Smoke test by installing into a scratch directory**

```bash
mkdir -p /tmp/talerid-smoke && cd /tmp/talerid-smoke
npm init -y
npm install @taler-id/oauth-client@0.1.0
node -e "const m = require('@taler-id/oauth-client'); console.log(typeof m.TalerIdClient);"
node -e "const m = require('@taler-id/oauth-client/react'); console.log(typeof m.TalerIdProvider, typeof m.useTalerIdAuth);"
```

Expected: each `node -e` prints `function`. The CJS resolution works, the ESM types are inferred via `exports` map, and the `/react` subpath works.

- [ ] **Step 7: Create a GitHub release**

```bash
cd ~/taler-id-sdk-js
gh release create v0.1.0 --title "v0.1.0 — initial release" --notes-file - <<'EOF'
First public release of `@taler-id/oauth-client`.

- Browser SPA SDK with Authorization Code + PKCE redirect flow
- Vanilla TypeScript core + opt-in React adapter (`@taler-id/oauth-client/react`)
- sessionStorage default (configurable to local / memory)
- On-demand token refresh with concurrent-call guard
- Single `TalerIdAuthError` with `code` discriminator

See [README](https://github.com/dvvolkovv/taler-id-sdk-js#readme) and the [integration guide](https://id.taler.tirol/oauth-guide.html).
EOF
```

Expected: GitHub release page is visible at `https://github.com/dvvolkovv/taler-id-sdk-js/releases/tag/v0.1.0`.

- [ ] **Step 8 (contingent): Manual e2e test against staging**

Optional but strongly recommended before announcing the release. Open the React example:

```bash
cd ~/taler-id-sdk-js/examples/react-vite
pnpm install
pnpm dev
```

Visit the URL Vite prints (default `http://localhost:5173`). Click "Sign in with Taler ID". Complete login on the staging Taler ID page. After redirect back, the page shows the user's `sub`. Click logout — the page reloads to the unauthenticated state.

If anything fails, file a follow-up task and consider whether v0.1.0 needs to be unpublished or superseded by v0.1.1.

## Out of Scope — Do Not Do

- **PROD deployment of any SDK consumer** — the demo controller rewrite (`~/taler-id/src/demo/`) is a separate follow-up commit, not part of this plan.
- **Vue / Svelte / Solid framework adapters** — vanilla SDK is the building block.
- **`<TalerIdLoginButton />` component** — Phase 0 ships HTML/CSS at `/brand`; duplicating in JSX is rejected.
- **Server-side Node.js variant** — `openid-client` already exists; the README links to it.
- **Background refresh timers** — refresh on demand only.
- **Popup login flow** — only redirect.
- **Promoting to v1.0.0** — wait until at least one external integrator validates the API in production.
- **Touching `~/taler-id`** — this plan operates entirely inside `~/taler-id-sdk-js`. The Taler ID backend repo is unrelated.
